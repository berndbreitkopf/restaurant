Arbeite diese Punkte vollständig ab:

1. Backend-Zugang / Admin sauber fertigstellen
- Prüfe, wie der Admin-Zugang aktuell umgesetzt ist.
- Stelle sicher, dass /admin wirklich funktioniert.

- Erstelle bei Bedarf eine Admin-Login-Seite unter /admin/login.
- Sorge dafür, dass nicht eingeloggte Nutzer sauber auf die Login-Seite umgeleitet werden.
- Erstelle eine klare Session-Prüfung.
- Admin-Logout einbauen.
- Falls noch kein Admin-User existiert, dokumentiere im Projekt klar, wie ein erster Admin angelegt wird.
- Zeige im Admin-Dashboard eindeutig, ob der Nutzer eingeloggt ist.
- Admin komplett auf Deutsch.

2. Alle restlichen grünen Elemente vollständig entfernen
- Durchsuche das gesamte Projekt nach allen grünen Farbresten.
- Ersetze alle hardcodierten Tailwind-Klassen wie bg-green-*, text-green-*, border-green-*, ring-green-*, from-green-*, to-green-*, emerald-*, lime-*, teal-*.
- Verwende stattdessen konsequent die bestehende warme Braun-/Gold-Farbwelt.
- Primärfarbe: warmes Espresso-Braun
- Akzent: Antikgold / Beige / Creme
- Dunkles Braun für Footer, Admin-Sidebar, Overlays
- Stelle sicher, dass wirklich keine grüne Hauptanmutung mehr vorhanden ist.

3. Getränkekarte deutlich hochwertiger machen
- Die Getränkekarte soll visuell stark verbessert werden.
- Füge passende hochwertige Bilder der Getränke ein, soweit von der Quellwebsite vorhanden.
- Nutze vorhandene Bilder der Quellseite lokal im Projekt.
- Wenn auf der Quellseite keine passenden Getränkebilder in ausreichender Qualität vorhanden sind, nutze die besten vorhandenen thematisch passenden Bilder aus den vorhandenen Assets.
- Baue eine hochwertige Getränkekarten-Seite mit:
  - Heißgetränke
  - Kaltgetränke
  - Biere
  - Aperitifs
  - Weine
  - traditionelle Kaffeezeremonie als optisches Highlight
- Nutze moderne Karten mit Bild, Titel, Beschreibung, Preis.
- Mobile Darstellung sauber optimieren.

4. Speisekarte mit hochwertigen Gerichtsbildern aufwerten
- Die Speisekarte soll deutlich hochwertiger wirken.
- Nutze lokale Bilder in hoher Qualität.
- Füge bei möglichst vielen Gerichten passende Food-Bilder ein.
- Falls mehrere Bilder vorhanden sind, wähle die beste Qualität.
- Speisekarte modern als Karten-/Grid-System darstellen.
- Kategorien klar strukturieren.
- Preisgestaltung sauber hervorheben.
- Optional Badges wie vegetarisch, vegan, scharf verwenden.
- Ladezeiten trotzdem performant halten.

5. Drei Tagesgerichte anlegen und sauber darstellen
- Lege direkt 3 Tagesgerichte im System an.
- Sie sollen auf /tageskarte sichtbar sein.
- Heutige Tageskarte prominent oben darstellen.
- Nutze moderne Kartenoptik mit Bild, Name, Beschreibung, Preis und Datum.
- Tagesgerichte sollen im Backend bearbeitbar, löschbar und neu anlegbar sein.
- Falls im Seed noch nichts vorhanden ist, lege 3 sinnvolle äthiopische Tagesgerichte als Startdaten an.
- Im Admin soll es eine klare CRUD-Verwaltung dafür geben.

6. Events logisch und hochwertig anzeigen
- Überarbeite die Event-Darstellung.
- Events sollen nicht nur als einfache Textblöcke erscheinen.
- Baue eine saubere Event-Sektion mit Karten.
- Jede Event-Karte soll enthalten:
  - Bild
  - Titel
  - kurzer Beschreibungstext
  - Wochentag oder Datum
  - optional Badge wie „Neu“, „Wiederkehrend“, „Live“
- Nutze vorhandene Event-Inhalte wie Eltern-Kind-Treffen, Feiertags-Brunch, Live Musik.
- Diese sollen auf der Startseite attraktiv sichtbar sein.
- Zusätzlich im Admin editierbar machen.
- Event-CRUD im Admin prüfen und vervollständigen.

7. Social-Media-Posting für Instagram, Facebook und TikTok sauber vorbereiten
- Prüfe den aktuellen Stand des Social-Media-Backends.
- Erstelle bzw. vervollständige einen Admin-Bereich für Social Posts.
- Dort soll man:
  - Bild/Video hochladen
  - Caption eingeben
  - Plattformen auswählen: Instagram, Facebook, TikTok
  - sofort posten oder planen
- Speichere Beiträge in social_posts.
- Erstelle eine saubere API-Route für Buffer-Integration.
- Route soll robust sein, TypeScript-sicher und sauberes Error Handling haben.
- Prüfe die ENV-Namen:
  - BUFFER_ACCESS_TOKEN
  - BUFFER_FACEBOOK_PROFILE_ID
  - BUFFER_INSTAGRAM_PROFILE_ID
  - BUFFER_TIKTOK_PROFILE_ID
- Zeige im Admin an, ob Buffer korrekt konfiguriert ist oder nicht.
- Wenn Buffer nicht konfiguriert ist, soll die UI verständlich darauf hinweisen.
- Ziel: Backend ist strukturell fertig für gleichzeitiges Posten auf Instagram, Facebook und TikTok.
